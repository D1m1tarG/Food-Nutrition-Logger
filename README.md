Food Nutrition Logger (v1.0)
Система за управление на хранителни режими и анализ на макронутриенти
1. Общ преглед 

Food Nutrition Logger е специализирано приложение, изградено на принципа на Data-Driven архитектурата. То позволява на потребителите да менажират каталог от хранителни продукти и да проследяват своята консумация в реално време. 

2. Системна Архитектура

Приложението е разделено на три основни логически нива:

Presentation Layer (Интерфейс): Изграден с Tkinter, осигуряващ събитийно-ориентирано управление (Event-driven) чрез графични компоненти (Tabs, Entries, Buttons).

Logic Layer (Приложна логика): Python скрипт, който обработва потребителските данни, извършва валидация и управлява потока от информация.

Data Layer (База данни): MongoDB сървър, съхраняващ информацията в гъвкав BSON формат.

3. Дизайн на базата данни

Използва се нерелационен модел, за да се осигури скалируемост и бързина при агрегирането.

Колекции и Схеми:
foods (Каталог):

name (String): Уникален идентификатор (Indexed).

calories, protein, carbs, fats (Double): Стойности за 100g.

daily_log (Хранителен дневник):

date (String/Date): Дата на записа (Indexed).

food_name (String): Връзка към каталога.

grams (Double): Количество.

macros (Object): Денормализирани данни за бърз достъп 

4. Техническа имплементация 

Агрегационен Пайплайн (Aggregation)

За получаване на дневния отчет се използва aggregate функцията, която изпълнява изчисленията на ниво база данни, намалявайки натоварването на клиента:

$match: Редуцира набора от данни до конкретна дата.

$group: Обединява всички записи, сумирайки числовите полета чрез акумулатора $sum.

$push: Трансформира редовете от базата в списък (Array) за детайлно изобразяване в интерфейса.

Оптимизация и Индексиране

За постигане на висока производителност (O(log n) сложност при търсене), са дефинирани следните индекси:

db.foods.create_index("name", unique=True) - Предотвратява колизии в данните.

db.daily_log.create_index("date") - Оптимизира филтрирането при големи обеми от записи.

5. Инструкции за разгръщане (Deployment)

Предварителни изисквания:
MongoDB Community Server (v5.0 или по-нова).

Python 3.8+.

Инсталиране на зависимости:

Bash

pip install pymongo

Стартиране:

Уверете се, че MongoDB сървърът е активен (порт 27017).

Стартирайте приложението: 

python nutrition_logger.py.

6. Валидация и Сигурност

Error Handling: Вградена проверка за тип данни при вход (Input Validation). Използване на try-except блокове за предотвратяване на грешки от тип ValueError.

Connection Timeout: Автоматично прекратяване на заявката при липса на връзка със сървъра (2000ms timeout).
